# Writer Skill

使用自定义写作风格生成或改写文章。

## 风格文件

风格文件存放在 `references/` 目录下，每个风格一个 `.md` 文件。

## 核心功能

### 1. 列出可用风格

读取 `references/` 目录，列出所有风格文件及其简介。

### 2. 用指定风格写作

1. 根据用户指定的风格名，匹配 `references/` 下的文件（匹配规则见下方）
2. 读取匹配到的风格文件到上下文
3. 按照风格文件中的指导进行写作
4. 直接输出文章内容，不要额外解释

**风格名匹配规则**（按优先级）：
1. 精确匹配文件名（不含 `.md`）
2. 忽略大小写匹配
3. 空格与连字符互换匹配（"Dan Koe" <-> "dan-koe"）
4. 匹配文件内 `# 标题` 中的风格名

### 3. 创建新风格

在 `references/` 目录下创建新的 `.md` 文件，参考现有风格的结构：

```markdown
# {风格名称}

**一句话风格描述。**

你是一位融合了 {风格名称} 写作精髓的作者。[角色定位描述]。
请将用户的内容用 {风格名} 风格重新演绎，直接输出文章内容，不需要其他说明。

## 核心理念
- ...

## 写作风格
**语气**：...
**视角**：...

## 文章结构
### 开头
### 主体
### 结尾

## 格式规范
- **粗体**：用于...
- *斜体*：用于...

## 语气要求
✅ 要做到：...
❌ 要避免：...

## 示例（可选）
### 标题公式（可选）
### 金句模板（可选）
```

## 使用示例

| 用户说 | 行为 |
|-------|-----|
| "列出写作风格" | 列出所有可用风格 |
| "用 Dan Koe 风格写一篇关于拖延的文章" | 匹配 `dan-koe.md`，生成文章 |
| "用 dan-koe 风格改写这段内容" | 匹配 `dan-koe.md`，改写内容 |
| "创建一个鲁迅风格" | 在 `references/` 创建 `lu-xun.md` |
